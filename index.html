Our desktop.css stylesheet uses the media query screen and (min-device-width:768px).

min-device-width tells us the available space of the device, while queries such as width (or min/max-width) tell us the width of the content.

Something to be aware of (as noted by QuirksMode):

Some mobile browsers, notably Safari iPhone, have a default layout viewport of around 850 to 1000 pixels — much larger than the device width. (Why? In order to accomodate desktop sites whose developers did not test on mobile. Such sites usually stretch to roughly that width.)

So the default ('mobile.css') stylesheet is the one you see on the mobile device, and devices with larger viewports get the extras. The reason we set 768px for the minimum device width was mainly to accommodate the iPad and any similar tablets that appear in future. We decided that iPad users should get the full site experience, and as the iPad’s shorter screen side has a width of 768px, that's the figure we chose. Any device with a lower resolution gets the mobile version.

See also: http://www.broken-links.com/2011/02/21/using-media-queries-in-the-real-world/ which is a useful article on Mobile First approaches (inc. a technique for loading the correct sized images by having the server send empty src values and instead assigning the source to data-src and using JavaScript to filter through the images and replace the src value appropriately - yes this means devices without JavaScript don't get to see the images but then if they don't have JavaScript enabled then viewable content is an acceptable consolation)

We could use Respond.js to provide media query support to IE6-8 but that only works when your media queries are using min/max-width and so doesn't work with min/max-device-width.

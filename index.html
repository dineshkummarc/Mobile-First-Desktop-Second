Mobile First (Desktop Second)

Description

We have three stylesheets

  1. mobile (min-width:320px)
  2. tablet (min-width:480px) and (max-width:959px)
  3. desktop (min-width:960px)

Differences between min-width and min-device-width

min-device-width tells us the available space of the device, while queries such as width (or min/max-width) tell us the width of the content.

Max Width Images

The following image uses max-width:100%; to ensure it doesn't overflow the maximum width of its container. So on the desktop version you'll just see the image as it should be, but on the tablet or mobile version (where the image would normally be wider than the content area) you would then see it resize to fit 100% of its container.

Dynamic image loading

See also: http://www.broken-links.com/2011/02/21/using-media-queries-in-the-real-world/ which is a useful article on Mobile First approaches (inc. a technique for loading the correct sized images by having the server send empty src values and instead assigning the source to data-src and using JavaScript to filter through the images and replace the src value appropriately - yes this means devices without JavaScript don't get to see the images but then if they don't have JavaScript enabled then viewable content is an acceptable consolation)

But really, I don't see why you couldn't load small low-res images for mobile and then use the data-attribute technique with JavaScript to load high-res images for tablets and desktop?
